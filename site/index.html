Your team pointed its agents at one repo. Now they collide on the same files.

Git catches text conflicts at merge — after everyone's agents have already burned tokens doing the work. Two teammates, five agents each, one codebase: the real cost is wasted parallel effort discovered far too late.

Tower is the missing write-side coordination layer. Every agent declares what it's about to change; Tower detects semantic overlap across the whole team and stops the second agent before it spends a token. It sits above git and speaks MCP — any agent, any model.

The pattern

Claim → detect → clear or hold

STEP 01

Claim intent

Before editing, the agent calls claim_intent with the files and symbols it's about to touch.

STEP 02

Detect overlap

Tower compares the intent against every active claim and grades the overlap: hard, soft, or clear.

STEP 03

Clear or hold

No overlap → proceed. Hard conflict → the edit is held and the agent is told who holds it and why.

✔ CLEARED   or   ⛔ HOLD

Cleared for takeoff

Install in one line

# .mcp.json
{
  "mcpServers": {
    "tower": { "command": "npx", "args": ["-y", "tower-mcp", "serve"] }
  }
}

Then add to CLAUDE.md: "Before editing any file, call claim_intent. If a hard conflict returns, stop and ask." Want it enforced? Enable the PreToolUse hook. 

# ~/.cursor/mcp.json
{
  "mcpServers": {
    "tower": { "command": "npx", "args": ["-y", "tower-mcp", "serve"] }
  }
}
git clone https://github.com/Rohanxmalik/Tower && cd Tower
npm install && npm run build
node packages/cli/dist/index.js serve
